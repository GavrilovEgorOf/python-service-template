# Customize after clone

Step-by-step checklist for turning this template into your production service. Work top to bottom; each phase builds on the previous one.

**Audience:** remote EU/US senior backend engineers starting a new FastAPI microservice.

## Phase 0 — Bootstrap

- [ ] Clone the template (or your fork) into a directory named after your service.
- [ ] Create a virtualenv and install dev dependencies:

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -e ".[dev]"
```

- [ ] Copy environment file and start dependencies:

```bash
cp .env.example .env
docker compose up -d postgres redis
```

- [ ] Apply migrations and confirm the API starts:

```bash
alembic upgrade head
uvicorn app.main:app --reload
```

- [ ] Hit liveness and readiness: `/health/live`, `/health/ready`.

## Phase 1 — Rename the service

Run once, immediately after clone:

```bash
python scripts/rename_service.py <your-service-name>   # e.g. billing-api
```

- [ ] Review the diff — the script rewrites all non-`.git` / non-`.venv` files.
- [ ] Update `pyproject.toml` `[project].authors` and description for your team.
- [ ] Set `.env` values:

```env
APP_NAME=<your-service-name>
OTEL_SERVICE_NAME=<your-service-name>
ENVIRONMENT=development
```

- [ ] Reinstall editable package after rename: `pip install -e ".[dev]"`.
- [ ] Commit the rename as a single initial commit in your service repo.

## Phase 2 — Replace the sample `Item` domain

The template ships a minimal CRUD example. Replace it with your domain using the same layered pattern.

### 2.1 Database model

- [ ] Edit or replace `app/db/models.py` — remove `Item`, add your entities.
- [ ] Keep models free of business logic; use SQLAlchemy 2.0 typed `Mapped` columns.

### 2.2 Schemas

- [ ] Replace `app/schemas/item.py` with DTOs for your domain (Create / Read / Update as needed).
- [ ] Use Pydantic `Field` constraints that match DB column limits.

### 2.3 Service layer

- [ ] Replace `app/services/item_service.py` with your business logic.
- [ ] Raise domain exceptions (`NotFoundError`, `ConflictError`, etc.); map them to HTTP in routes.
- [ ] Keep functions async and accept `AsyncSession` as the first argument after `self` (if class-based).

### 2.4 HTTP routes

- [ ] Replace `app/api/routes/items.py` or add new route modules under `app/api/routes/`.
- [ ] Register routers in `app/api/router.py`.
- [ ] Use appropriate status codes (201 create, 409 conflict, 404 not found).

### 2.5 Migrations

- [ ] Remove or supersede `alembic/versions/0001_create_items_table.py` if starting fresh on an empty DB.
- [ ] Generate a new migration:

```bash
alembic revision --autogenerate -m "create <your_table>"
alembic upgrade head
```

- [ ] **Review autogenerated SQL** — Alembic misses renames and some constraints.

### 2.6 Tests

- [ ] Delete or rewrite `tests/test_items.py`.
- [ ] Add **unit tests** for service functions (mocked session).
- [ ] Add **integration tests** for HTTP endpoints using the `client` fixture from `tests/conftest.py`.
- [ ] Keep `tests/test_health.py` unless health contract changes.

### 2.7 Cleanup

- [ ] Remove unused imports and dead files (`item.py`, `items.py`, etc.).
- [ ] Update README example `curl` commands and OpenAPI tags.

## Phase 3 — Enable authentication

The template is open by default. `.env.example` includes auth-related placeholders; wire them when you need protection.

### 3.1 Settings

- [ ] Extend `app/core/config.py` with settings from `.env.example`:

```python
auth_disabled: bool = False
api_key: str = "change-me"
api_v1_prefix: str = "/api/v1"
```

- [ ] Never commit real secrets; use your secret manager in production.

### 3.2 Dependencies

- [ ] Create `app/api/deps.py` with FastAPI dependencies, for example:

  - **API key** — `HTTPBearer` or header-based key check against `settings.api_key`.
  - **JWT / OAuth2** — validate tokens from your IdP (Auth0, Cognito, Keycloak); extract `sub` / scopes.
  - **Dev bypass** — when `settings.auth_disabled` is true, skip verification locally only.

- [ ] Apply dependencies at router level:

```python
router = APIRouter(prefix="/items", tags=["items"], dependencies=[Depends(require_auth)])
```

- [ ] Leave `/health/live` and `/health/ready` unauthenticated for orchestrators.

### 3.3 Tests

- [ ] Add integration tests for 401/403 without credentials and 200 with valid auth.
- [ ] Use `app.dependency_overrides` in tests to inject a fake principal.

## Phase 4 — Wire infrastructure

### 4.1 PostgreSQL

- [ ] Set `DATABASE_URL` in `.env` (async URL: `postgresql+asyncpg://...`).
- [ ] Confirm readiness probe passes (`/health/ready` runs `SELECT 1`).
- [ ] For managed Postgres (RDS, Cloud SQL, Neon): enable SSL params in the URL, tune pool size in `app/db/session.py` if needed.

### 4.2 Redis

- [ ] `REDIS_URL` is in `.env.example`; add a Redis client module when you need caching, rate limits, or idempotency keys.
- [ ] `docker compose` already runs Redis on port 6379.
- [ ] Use `.env` keys `CACHE_TTL_SECONDS` and `IDEMPOTENCY_TTL_SECONDS` as starting defaults for your implementation.

### 4.3 Observability

- [ ] **Structured logging** — set `LOG_JSON=true` in staging/production; keep `LOG_LEVEL=INFO` or `WARNING`.
- [ ] **OpenTelemetry** — enable when a collector is available:

```env
OTEL_ENABLED=true
OTEL_EXPORTER_ENDPOINT=http://otel-collector:4317
OTEL_SERVICE_NAME=<your-service-name>
```

- [ ] Local collector: `docker compose --profile observability up --build`
- [ ] Collector config: `deploy/otel-collector-config.yaml` — point exporters to your backend (Jaeger, Tempo, Datadog, etc.).

### 4.4 Docker image

- [ ] Build and run the full stack: `docker compose up --build`
- [ ] Production Dockerfile uses `uvicorn` without `--reload`; run migrations in your deploy pipeline or an init container, not only in dev compose.
- [ ] Pin base image digests in production CI/CD.

### 4.5 Deployment artifacts (your platform)

- [ ] Add Helm chart, Terraform module, or Pulumi stack for your cluster/cloud.
- [ ] Configure liveness → `/health/live`, readiness → `/health/ready`.
- [ ] Set resource requests/limits, HPA on CPU or RPS, and graceful shutdown (`terminationGracePeriodSeconds` ≥ in-flight request timeout).

## Phase 5 — Production checklist

Before first production deploy:

### Security

- [ ] `AUTH_DISABLED=false` in all non-local environments.
- [ ] Rotate `API_KEY` / JWT signing keys; load from secret manager, not plain env files in git.
- [ ] `DEBUG=false`, restrict CORS to known origins, disable public `/docs` if required by policy.
- [ ] Run dependency scanning in CI (`pip-audit` or GitHub Dependabot).

### Data

- [ ] Migrations tested on a staging DB clone; rollback plan documented.
- [ ] Backups and PITR enabled on managed Postgres.
- [ ] No destructive migrations without expand/contract strategy.

### Reliability

- [ ] Readiness fails when Postgres is unreachable (already default behavior).
- [ ] Timeouts and connection pool limits configured for expected load.
- [ ] Idempotency for write endpoints that external systems retry.

### Observability

- [ ] JSON logs shipped to your aggregator (CloudWatch, Loki, ELK).
- [ ] Traces exported with service name, environment, and version resource attributes.
- [ ] Alerts on readiness failures, error rate, and latency SLOs.

### CI/CD

- [ ] Pipeline runs `ruff`, `mypy`, `pytest --cov=app` (same as `.github/workflows/ci.yml`).
- [ ] Deploy only from tagged releases or protected branches.
- [ ] Smoke test health endpoints after deploy.

### Documentation

- [ ] README updated with your service purpose, runbook link, and on-call contact.
- [ ] ADR added under `docs/adr/` for non-obvious architectural choices.

## Quick reference — files to touch

| Goal | Primary files |
|------|-----------------|
| Rename | `scripts/rename_service.py`, then review full diff |
| Domain | `app/db/models.py`, `app/schemas/`, `app/services/`, `app/api/routes/` |
| Migrations | `alembic/versions/` |
| Auth | `app/api/deps.py`, `app/core/config.py`, route `dependencies=` |
| Config | `.env`, `.env.example`, `app/core/config.py` |
| Observability | `app/core/logging.py`, `app/core/telemetry.py`, `deploy/otel-collector-config.yaml` |
| Tests | `tests/unit/`, `tests/test_*.py`, `tests/conftest.py` |
